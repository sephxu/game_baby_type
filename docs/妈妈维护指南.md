# 妈妈维护指南

这份指南是给阿悠妈妈用的。日常使用不需要懂代码，也不需要把应用发布到网上。

## 日常打开

平时打开 `Game of Type.app` 就可以玩。应用会在本机自动启动一个本地服务，让图片、语音和新增单词功能都走同一套本地地址，避免直接打开 `index.html` 时出现语音不一致。

如果是在开发环境里测试，可以让 AI Coding 工具或懂技术的人运行：

```bash
npm run electron
```

## 添加新单词

1. 打开 `Game of Type.app`。
2. 点击右上角家长设置按钮。
3. 在 `Add Word` 输入一个英文单词。
4. 点击 `Add`。
5. 新单词会保存到 `data/pending-words.json`。

这里保存的只是“待补充单词”。完整闪卡还需要补充句子、图片、语音和词卡数据。

## 让 Codex 或 AI Coding 工具处理

添加完单词后，可以打开 Codex 或其他 AI Coding 工具，把下面这段话发给它：

```text
请读取 data/pending-words.json，把 pending 的单词补充为正式闪卡内容：
1. 为每个单词生成适合幼儿的英文短句；
2. 更新词卡数据；
3. 生成或接入对应图片；
4. 生成或更新语音资源和 manifest；
5. 跑 npm test、npm run build、npm run electron:check；
6. 确认通过后 git commit。
```

如果只是修改游戏规则，也可以直接描述需求，例如：

```text
请把挑战模式时间改成 45 秒，保持其他逻辑不变，完成验证并提交。
```

## 验证和提交

AI 修改完以后，应该至少运行：

```bash
npm test
npm run build
npm run electron:check
```

通过后再执行 `git commit`。提交信息不需要任何 AI 署名。

## 出问题时

- 如果新增单词失败：确认是从 `Game of Type.app` 或 `npm run electron` 打开的，而不是直接双击 `index.html`。
- 如果语音听起来不一致：优先确认不是 `file://` 方式打开。
- 如果 AI 不知道怎么做：把 `docs/ai-coding-prompts.md` 里的对应提示词复制给它。
